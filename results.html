Findings
Key Observations
PROGRAMMED I/O

Fastest for tiny data (<5MB) due to zero setup overhead. CPU utilization stays at ~68% regardless of data size — it never improves, making it unsuitable for large transfers.

INTERRUPT-DRIVEN I/O

CPU utilization grows with data size due to more interrupts and context switches. Optimal range is 10–80MB — frees CPU between transfers while keeping hardware cost reasonable.

DMA

CPU overhead stays near 0% regardless of data size. Total time is lowest for large transfers. Bus arbitration slightly reduces effective rate to ~97 MB/s, but still outperforms others at scale.
