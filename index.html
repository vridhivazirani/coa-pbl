Problem Statement

Why I/O Efficiency Matters

Efficient data transfer between the CPU, main memory, and input/output devices is essential for overall system performance in computer architecture. Different I/O data transfer techniques — Programmed I/O, Interrupt-Driven I/O, and Direct Memory Access (DMA) — vary in terms of CPU involvement, hardware complexity, and bus utilization.

Inefficient transfer methods can lead to increased CPU overhead, reduced throughput, and poor system responsiveness. This project aims to study, implement, and evaluate these I/O transfer techniques from a hardware architecture perspective — analyzing their operational mechanisms, performance impact, and suitability for different data workloads within a computer organization framework.

Reference

COA Glossary

Status Register
Holds flags (BUSY, READY, ERROR) that the CPU checks to know if an I/O device is ready for transfer.
Busy-Wait / Polling
The CPU continuously checks a device's status register in a loop until it is ready — wasting CPU cycles.
Interrupt Service Routine
A special function (ISR) executed by the CPU when it receives an interrupt signal from an I/O device.
Interrupt Vector Table
A table storing addresses of ISRs indexed by interrupt number, so the CPU knows where to jump.
Context Switching
Saving the current CPU state and loading another process's state — necessary when servicing an interrupt.
DMA Controller
A dedicated hardware unit managing data transfers between memory and I/O devices without CPU involvement.
Bus Arbitration
The process by which the DMA controller or CPU gains ownership of the system bus to perform a transfer.
Throughput
Amount of data successfully transferred per unit of time (MB/s) — a key system performance metric.
CPU Overhead
The fraction of CPU time consumed managing I/O instead of executing useful application code.

Research Approach

Project Methodology

01

Literature Review

Explored CPU, memory, and I/O structures. Reviewed PIO, Interrupt-Driven I/O, and DMA from foundational texts.

02

System Design

Created architectural diagrams for each I/O approach. Mapped data flow, control lines, and bus activity.

03

Implementation

Simulated data transfer for small, medium, and large sizes. Modeled CPU actions under each method in Python.

04

Performance Eval

Tracked CPU utilization, transfer speed, throughput (MB/s), and bus load across all three techniques.

05

Comparative Analysis

Studied efficiency, hardware requirements, and identified the best use cases for each transfer method.

What We Built

Yes — the theory of I/O transfer techniques is documented in textbooks. But this project is not a textbook. Here is what we built from scratch that did not exist before.

01
Original Python Simulation
We wrote a custom performance model with defined constants — transfer rate, interrupt overhead, context switch cost, DMA init time, and block size — and ran it across 7 data sizes.
Python · Quantitative Analysis
02
Live Interactive Simulator
We rebuilt the Python model in JavaScript connected to a real-time slider. Any data size from 1MB to 1GB updates all three methods simultaneously.
JavaScript · Real-time Computation
03
Animated Bus Visualizer
A step-by-step animation showing exactly what happens on the system bus for each I/O technique. This does not exist in any textbook or standard COA resource.
Animation · Hardware Modeling
04
4-View Dynamic Chart System
Built from scratch using HTML Canvas API — no library — rendering four performance views that update live based on selected data size.
Canvas API · Data Visualization
05
Smart Recommendation Engine
Logic that evaluates which I/O method is optimal for any given data size and explains why — applied analysis, not just theory.
Decision Logic · Applied COA
06
Full Multi-Page Web Platform
Designed and coded as a single self-contained multi-page web app — not a slide deck, not a copied resource. A working tool built for this subject.
HTML · CSS · JavaScript · No Frameworks
